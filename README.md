## Chatbot CRM Triggers

- After urgency: set priority flag.
- After contact info: auto-assign to intake rep, create "Call within 1 hour" task.

## Chatbot Human Takeover Points
- High urgency, high volume, or multi-location chains.
- Negative sentiment, legal questions, or sensitive data.
- Pricing-only conversation after two messages.
- Any mention of bad past agency experience.
- Any request for refund processing or assurance disputes.

## CRM Structure and UX
- Objects: Lead, Conversation, Call, Appointment, Campaign, Task, Consent Record.
- Lead fields: source, business type, locations, urgency, budget range, owner status, last touch, next action.
- Conversation fields: channel, transcript, sentiment tag, outcome.
- Call fields: recording link, disposition, callback flag, TCPA consent.
- Appointment fields: date/time, owner, status, notes.
- Status pipeline: New -> Contacted -> Qualified -> Booked -> Active -> Won/Lost -> Nurture.
- Timeline view: every event in order with source labels and next action pinned.
- Daily view: "Today's Calls," "Overdue Follow-ups," "Booked This Week."
- Automations: duplicate detection by phone/email; auto-close after 30 days of inactivity; escalation to manager after two missed follow-ups.
- Security embedded: role-based access, field-level masking, audit trails on edits and exports, encryption at rest/in transit, short-lived session tokens, IP allowlist option, consent log immutable.
- Assurance fields: assurance status, assurance window start/end, primary engagement goal, engagement signals log, eligibility checklist, refund cap, results achieved flag, refund decision.
- Assurance automations: day-7 and day-10 alerts, auto-create refund review task if no engagement signals logged, manager approval required to finalize refunds.

## Embedded Security Safeguards (Applied Across System)
- Forms: CSRF protection, input validation, honeypot + rate limits, signed webhooks, encryption on submit.
- Chatbot: abuse throttling, transcript redaction for sensitive fields, origin checks.
- CRM: least-privilege roles, field-level permissions, full audit logs, data retention controls, automatic lock on suspicious logins.
- Calls and recordings: access restricted by role, time-limited links, consent records displayed before playback.
- Reporting: aggregated views default, PII only when necessary, watermark on exports.
- Assurance safeguards: eligibility checklist locked, refund approvals require dual sign-off, refund actions audit-logged, abuse detection on repeated assurance claims.

## Conversational Control Layer (Applied Across Channels)
- Core objective: Maintain forward momentum and guide toward booking without pressure.
- Default frame: Assume alignment after intent and offer choices instead of decisions.
- Booking rule: Never ask "Would you like to book?" always ask "When should we book?"
- Time options: Present two time windows to reduce decision fatigue.
- Fail-safe exit: If declined, leave the door open without chasing.
