# App Wireframe: Markter Ops (Internal App)
Navigation: Leads, Conversations, Calls, Appointments, Campaigns, Reports, Team, Settings

- Screen: Login. Headline: Welcome back. Subhead: Secure access to Markter Ops. Body: Sign in and confirm your device. Primary CTA: Sign in. UX intent: Fast entry with security. Embedded safeguard: MFA required, device trust, session timeouts.
- Screen: Leads Inbox. Headline: New leads today. Subhead: Every lead has a next step. Body: List view with status, source, last touch, next action. Primary CTA: Open lead. UX intent: Zero-click triage. Embedded safeguard: PII masked in list, reveal on open.
- Screen: Lead Profile. Headline: Lead record. Subhead: Full timeline, no gaps. Body: Contact details, source, tags, status history, conversation log. Primary CTA: Add note | Call now | Schedule. UX intent: Single-screen control. Embedded safeguard: Field-level permissions, audit log on edits.
- Screen: Conversation Timeline. Headline: Conversation history. Subhead: Text, chat, and call in one stream. Body: Chronological timeline with tags and outcomes. Primary CTA: Send message. UX intent: Context without switching tabs. Embedded safeguard: Secure message templates, link tracking with expiration.
- Screen: Calls Inbox. Headline: Calls that need attention. Subhead: Missed and returned calls. Body: Callback queue, recordings, disposition. Primary CTA: Call back. UX intent: Protect revenue from missed calls. Embedded safeguard: Call recordings access by role only.
- Screen: Appointment Scheduler. Headline: Booked and pending. Subhead: Keep the pipeline moving. Body: Calendar with availability, confirmations, reminders. Primary CTA: Book appointment. UX intent: Reduce scheduling friction. Embedded safeguard: Write access limited to schedulers; cancellations logged.
- Screen: Campaign Overview. Headline: Campaign health. Subhead: What's working today. Body: Spend, calls, orders, CPL/CPA, alerts. Primary CTA: View details. UX intent: Snapshot decisions. Embedded safeguard: Change approvals for budgets.
- Screen: Reports. Headline: Proof of performance. Subhead: Funnel from click to close. Body: Attribution, conversion rate, outcome by source. Primary CTA: Export report. UX intent: Transparency. Embedded safeguard: Export logs + watermarking.
- Screen: Team & Permissions. Headline: Roles and access. Subhead: Least-privilege by default. Body: Roles, access scopes, device list. Primary CTA: Invite user. UX intent: Fast onboarding with safety. Embedded safeguard: IP allowlist option, forced MFA.
- Screen: Settings. Headline: System controls. Subhead: Secure by default. Body: Data retention, consent records, webhook signing keys. Primary CTA: Save changes. UX intent: Minimize admin friction. Embedded safeguard: Admin actions require re-auth.
- Screen: Assurance Tracker. Headline: 10-Day Results Assurance. Subhead: Transparent progress, day by day. Body: Countdown to day 10, primary engagement goal, engagement signals, eligibility checklist, refund cap status. Primary CTA: Confirm eligibility | Mark results achieved. UX intent: Keep assurance visible and accountable. Embedded safeguard: Only managers can approve refunds or mark assurance completion.
